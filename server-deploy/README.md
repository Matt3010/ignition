# Ignition server deployment

Copy this entire folder to the server, for example to `~/road` or `/opt/ignition`.
No application source code, Node.js, npm, or local build is required.

## First installation

```bash
cp .env.example .env
nano .env
```

Set at least:

```env
IGNITION_IMAGE=ghcr.io/<github-owner>/<repository>:latest
OSM_REGIONS=italy
```

For a private GHCR package, authenticate with a GitHub Personal Access Token classic that has `read:packages`:

```bash
read -s GHCR_TOKEN
echo "$GHCR_TOKEN" | docker login ghcr.io -u <github-user> --password-stdin
unset GHCR_TOKEN
```

Start the stack:

```bash
docker compose up -d
```

The project automatically initializes `valhalla.json`. If no Valhalla tiles exist, the maintenance service performs the initial OSM refresh immediately.

## Updates

After GitHub Actions publishes a new image:

```bash
./update.sh
```

## Useful checks

```bash
docker compose ps
docker compose logs -f backend
docker compose logs -f osm-refresh
docker compose logs -f valhalla
docker compose logs -f postgres
```

## Files to keep on the server

```text
docker-compose.yml
.env
update.sh
```

Keep the `data/`, `reports/`, and Docker volumes generated by Compose. Do not delete them during normal updates.
